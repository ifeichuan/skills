## Top20 速记补丁

下列速记来自同源 QA 的 `recommended_answer`，只用于讲解前的短提示；完整答案仍必须回源原记录，且个别自动归类可能跨领域，必要时标 `uncertain`。

- **vue2和vue3的区别**：Vue 2 主要用 `Object.defineProperty` 劫持已有属性；Vue 3 用 `Proxy` 代理对象，新增/删除属性与 Map/Set 支持更自然。
- **自我介绍**：按背景、实习/项目、代表成果、岗位匹配组织，保持真实具体，避免只报技术栈。
- **进程和线程的区别**：进程有独立地址空间、偏资源隔离；线程共享进程资源、偏 CPU 调度，线程切换轻但需处理竞争。
- **性能优化**：先用 Lighthouse/Web Vitals/性能面板定位，再从资源、渲染、运行时优化，并给出指标前后对比。
- **http和https的区别**：HTTPS 在 HTTP 上加入 TLS，提供加密、身份认证与完整性保护；证书用于验证身份，对称加密承担数据传输。
- **事件循环机制**：同步代码结束后清空微任务，再取下一个宏任务；浏览器还要结合渲染机会，微任务递归可能阻塞更新。
- **vue2和vue3的区别**：Vue 2 响应式有新增/删除属性与数组索引边界；Vue 3 的 Proxy 覆盖整个对象并支持更完整的数据结构。
- **vite和webpack的区别**：Webpack 开发时以构建依赖图和 bundle 为中心；Vite 开发态基于原生 ESM 按需加载，生产构建再打包。
- **get和post的区别**：GET 侧重获取、通常可缓存且要求幂等；POST 侧重提交副作用，缓存与幂等取决于语义和服务端设计。
- **==和===的区别**：`==` 允许类型转换后比较；`===` 不做隐式类型转换，工程回答应补充边界与实际场景。
- **深拷贝和浅拷贝的区别**：浅拷贝只复制第一层引用；深拷贝递归复制嵌套结构，还需考虑循环引用、Symbol、Date/Map/Set 等。
- **v-if和v-show的区别**：`v-if` 按条件创建/销毁，适合低频切换；`v-show` 始终渲染只切换 display，适合高频切换。
- **computed和watch的区别**：computed 是带缓存的同步派生值；watch 监听变化执行副作用，适合异步请求、埋点和联动。
- **浏览器缓存机制**：强缓存命中时不请求服务器；过期后通过 ETag/Last-Modified 协商，未变化返回 304。
- **浏览器渲染原理**：从 URL/缓存到 DOM、CSSOM、Render Tree，再经 Layout、Paint、Composite；脚本和 CSS 可能阻塞解析/渲染。
- **防抖和节流的区别**：防抖把连续触发合并为最后一次，节流限制时间窗口内最多执行一次；输入搜索和滚动监听是典型场景。
- **闭包是什么？**：闭包让函数在外层执行结束后仍访问词法作用域，适合私有状态/柯里化，也可能因长期引用造成内存问题。
- **ref和reactive区别**：`ref` 适合包装值且通过 `.value` 访问；`reactive` 代理对象，使用时需注意解构导致响应性丢失等边界。
- **vue响应式原理**：核心是依赖收集与触发更新：读取时 track，写入时 trigger；Vue 2/3 的底层拦截机制不同。
- **tcp和udp的区别**：TCP 提供连接、可靠、有序传输；UDP 无连接、开销小、低延迟但可靠性需业务层补足。
